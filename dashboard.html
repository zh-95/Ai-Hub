BLOCCANTE

1. Persistenza settings

Usare electron-store (npm, schema validation, ~120KB extra) o JSON manuale con fs.promises (zero-dep, ma più codice)?

Sblocca Worker Core su F3 (storage/settings.ts). Senza questa decisione il Worker Core non può iniziare.

BLOCCANTE

2. Drag finestra frameless

Usare -webkit-app-region: drag CSS (blocca click sulle drag region) o ipcRenderer.send('window:start-drag') + coordinate mouse?

Blocca sia Worker UI (F1) che Worker Core (F2). Devono essere allineati. Decisione cross-modulo — va presa dal proprietario.

DECISIONE

3. Cifratura API key

Base64 (reversibile, zero-deps, dichiaratamente temporaneo) o safeStorage Electron (cifra con OS, ma richiede app firmata su Windows)?

In v1 l'app non è firmata. Base64 è la misura consapevole temporanea. Confermare o cambiare approccio.

DECISIONE

4. Provider CLI e prompt AI

CLI: comando come stringa unica ("claude --print") o array (["claude","--print"])? Prompt AI: hard-coded o configurabile dall'utente?

Array è più sicuro (execFile vs exec). Prompt configurabile è più flessibile ma richiede UI extra. Sblocca Worker AI su F5.

DECISIONE

5. Parser .md e file picker

Parser: regex semplici (zero-dep, sufficienti per struttura fissa) o libreria marked (~60KB, più robusta)? File picker: IPC dialog.showOpenDialog o <input webkitdirectory>?

Regex raccomandate per v1. IPC raccomandata per il picker (più robusto su directory grandi). Confermare per sbloccare F4 e F1.

PRIORITA'

6. Prossimo lavoro tecnico

Sequenza consigliata dopo le risposte: 1» avviare Worker Core su F2, 2» avviare Worker UI su F1, 3» avviare Worker AI su F5, 4» Worker Core su F3+F4.

F0 è solida. Worker Core e Worker UI possono partire in parallelo una volta risolti i 2 BLOCCANTI. Worker AI dipende solo da decisioni minori.
